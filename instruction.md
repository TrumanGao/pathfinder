## 任务总览

### Phase 1: 前期 - 契约先行与立项（约占总周期 20%）

1. 定义图结构接口

- 工作量：轻。
- 任务：起草并提交全英文的 Proposal（借助 AI 极速完成）。同时，写出最基础的 Node 和 Edge 等的 Java 类定义，扔到群里让大家遵守，不需要写任何业务逻辑。

2. 定义前后端交互契约（JSON Mock）

- 工作量：轻。
- 任务：设计 RESTful API 的返回格式。最关键的是：要手写一份假的 JSON 数据（Mock API request Data）。比如伪造一份 Dijkstra 跑完后的路径坐标和耗时。这样其他人可以直接拿着假数据去开发前端。

3. 双端脚手架搭建

- 工作量：轻。
- 任务：初始化 Spring Boot 项目，配置好跨域；用 Vite 初始化 React 项目。把这个空壳子推送到代码仓库，大家基于这个骨架开工。

### Phase 2: 中期 - 核心算法与视觉开发（约占总周期 50%）

1. 经典算法：Dijkstra 算法 + 简易测试图

- 工作量：中。
- 任务：实现 Dijkstra，并在后端硬编码 1-2 个极简的二维网格数组（比如 0 代表平地，1 代表墙）转化为图结构，供大家跑算法测试。

2. 启发算法：A\* 算法实现

- 工作量：中。
- 任务：实现 A\* 算法，写好曼哈顿距离的启发函数。

3. 算法与动画：双向 BFS 算法

- 工作量：中。
- 任务：后端实现双向 BFS。

4. 前端核心动画渲染

- 工作量：中偏重。
- 任务：利用假数据，写出 `setTimeout` 的渲染逻辑，把“探索过程”和“最终路径”在页面上用不同的颜色画出来。

### Phase 3: 后期 - 联调、AI 注入与交付（约占总周期 30%）

1. 后端汇总：合并算法与压测

- 工作量：轻。
- 任务：把大家写的三个算法类汇总到一个 Controller 里。确保在同一个起点和终点下，三个算法能正确输出各自的耗时和遍历节点数。

2. AI 升华：大模型 API 接入

- 工作量：中。
- 任务：拿到 A 汇总好的真实测试数据后，调用大语言模型 API（如 Gemini/ChatGPT），生成一段针对这三次算法运行结果的优劣势分析文本，并作为 JSON 的一部分返回给前端。

3. 终极组装：真实联调与手动打包

- 工作量：中。
- 任务：前端去掉假数据（Mock），换成请求后端的真实接口。处理可能出现的细微 Bug。最后拷贝打包出最终用于演示的 `.jar` 包。
